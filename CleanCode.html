• Poor variable Naming Conventions -->
    • camelCase for params, local variables,
    • snake_case for private variables
    • PascalCase for methods and properties
  • Poor Method Naming Conventions -->
    • Method Should explain what it does
    • If it is function that returns values it should start with "Get" or "Retieve" or "Return"
  • Long Parameter List -->
    • Long method paramter list should be replaced with class or structure entity
  • Variable Declarations on the Top
  • Magic Numbers -->
    • Avoid using magic numbers and use enums or constant instead
  • Nested conditional -->
    • Simplify code by reducing unnecessary if/else clauses
  • Duplicate Code -->
    • Remove duplications
    • Move duplicated code to resusable methods (mostly private)
  • Blob classes -->
    • Split classes into multiple classes
    • When having huge methods - extract it's contect to classes
  • Switch statements -->
    • when switch statements use type as parameter -> replace with abstract classes
    • use enums when possible
  • Comments -->
    • Code should be descriptive not to require comments
    • When writing comments write HOW not WHAT
  • Long methods -->
    • 5-10 lines, 20 lines max. Otherwise hard to name
    • Should be splited to several smaller methods so code is written as well-written prose
    • Or split into class
